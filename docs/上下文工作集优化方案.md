# 上下文工作集实现合同

## 1. 文档角色

本文是 nanobot 当前短期上下文主链的权威说明，描述运行时到底读取什么、裁剪什么、以及哪些旧路径已经退出主状态职责。

本文不再讨论“准备怎么改”，只描述当前实现的有效合同。

## 2. 当前主链

当前短期上下文的主链由结构化状态驱动，核心对象是：

1. `WorkingSetSnapshot`
2. `Recent Raw Turns`
3. `TurnCapsule`
4. `ArtifactRecord`

其中，真正的稳定主输入是 `WorkingSetSnapshot`。`Recent Raw Turns` 只提供最近合法原始尾部，`TurnCapsule` 和 `ArtifactRecord` 作为可选补充层进入 prompt。

当前实现中：

- `ContextBuilder.build_messages()` 接收显式结构化输入，而不是把 `history` 当成唯一主入口。
- `AgentLoop` 在发起模型调用前读取最新稳定 `WorkingSetSnapshot`，再补入最近 raw turns。
- `AgentLoop.finalize_turn()` 在 turn 提交成功时写入新的 capsule、working set、response 和 manifest。
- `AutoCompact` 只生成 `autocompact_candidate` 快照，不推进 `latest-working-set.json`。
- `Dream` 只消费 committed turn 的结构化产物，并明确把 `working/CURRENT.md` 视为镜像输出。

## 3. Prompt 主输入

### 3.1 唯一稳定读源

运行时稳定短期状态的唯一读源是 `latest-working-set.json` 指向的 `WorkingSetSnapshot`。

这意味着：

- 当前任务目标、阶段、相关 capsule/artifact 引用，首先从 working set 读取。
- `session.messages` 尾部不再承担“当前任务主记忆”的职责。
- `working/CURRENT.md` 即使出现在 system prompt 中，也只是一份镜像文本，不是主真相源。

### 3.2 补充读源

当前 prompt 仍会读取以下补充层：

- `Recent Raw Turns`：来自 `Session.get_history()` 的最近合法原始尾部
- `Selected TurnCapsules`：当调用方提供时作为补充区块注入
- `Selected Artifact Render`：当调用方提供时以 prompt-safe render 注入

这些补充层都不能覆盖 `WorkingSetSnapshot` 的稳定职责。

## 4. Prompt 组装合同

`ContextBuilder.build_messages()` 的结构化输入如下：

```python
def build_messages(
    self,
    *,
    working_set: dict[str, Any] | None = None,
    recent_raw_turns: list[dict[str, Any]] | None = None,
    selected_capsules: list[dict[str, Any]] | None = None,
    selected_artifacts: list[dict[str, Any]] | None = None,
    current_message: str,
    ...
) -> list[dict[str, Any]]:
    ...
```

当前固定组装顺序是：

1. `System Prefix`
2. `WorkingSetSnapshot`
3. `Recent Raw Turns`
4. `Selected TurnCapsules`
5. `Selected Artifact Render`
6. `Current User Message`

当前固定裁剪顺序是：

1. 低相关 capsule
2. 低相关 artifact render
3. 较旧 raw turns
4. working set 里的 `budget_hints`
5. 不裁 system prompt
6. 不轻易裁 stable working set

## 5. 核心对象

### 5.1 WorkingSetSnapshot

`WorkingSetSnapshot` 是当前运行时主工作内存对象。当前实现使用以下字段语义：

```python
WorkingSetSnapshot = {
    "session_key": str,
    "version": int,
    "source_turn_id": str | None,
    "source_revision": int | None,
    "is_stable": bool,
    "published_by": str,  # agent_loop | autocompact_candidate
    "active_task": str | None,
    "task_stage": str | None,
    "active_goals": list[str],
    "open_loops": list[str],
    "last_user_focus": str | None,
    "relevant_capsule_refs": list[str],
    "relevant_artifact_refs": list[str],
    "budget_hints": dict,
    "source_turn_ids": list[str],
    "created_at": str,
}
```

约束：

- 版本只追加，不原地覆写。
- `latest-working-set.json` 只能指向稳定快照。
- `published_by=autocompact_candidate` 只表示候选快照，不能自动晋升为最新稳定工作集。

### 5.2 ArtifactRecord

`ArtifactRecord` 承接工具结果及其可复用投影，避免 prompt 直接回灌原始大结果。

```python
ArtifactRecord = {
    "artifact_id": str,
    "session_key": str,
    "turn_id": str,
    "tool_call_id": str | None,
    "declared_revision": int,
    "source_type": str,
    "source_input": dict,
    "raw_ref": str,
    "digest": str,
    "size_chars": int,
    "freshness_policy": str,
    "content_version": str | None,
    "invalidated_by": list[str],
    "created_at": str,
}
```

约束：

- `read_file` artifact 带文件版本信息。
- `exec` artifact 带命令签名和工作目录。
- stale revision 的结果可以保留为审计对象，但不能进入当前稳定提交。
- prompt 默认读取 render 视图，而不是 raw payload。

### 5.3 TurnCapsule

`TurnCapsule` 是稳定完成后的回合级结构化结论。

```python
TurnCapsule = {
    "capsule_id": str,
    "turn_id": str,
    "session_key": str,
    "source_revision": int,
    "user_goal": str,
    "assistant_intent": str,
    "decisions": list[str],
    "outcomes": list[str],
    "open_questions": list[str],
    "artifact_refs": list[str],
    "next_expected_action": str | None,
    "capsule_version": int,
    "created_at": str,
}
```

约束：

- capsule 由 turn 稳定提交链路产出。
- `WorkingSetSnapshot`、Dream、AutoCompact 都只消费稳定 capsule 引用。

### 5.4 Recent Raw Turns

`Recent Raw Turns` 是补充层，不是主状态层。

当前实现里它主要来自 `Session.get_history()`，保留原因有两个：

- 给尚未被结构化对象覆盖的最近细节提供最小原始上下文
- 维持工具调用合法边界和最近对话连续性

约束：

- 只作为补充输入
- 优先级低于 `WorkingSetSnapshot`
- 不能重新升级成“短期主记忆”

## 6. 读写边界

### 6.1 AgentLoop

`AgentLoop` 是 active turn 的状态推进者：

- 读取 `latest-working-set.json`
- 读取 active/latest turn index
- 负责 `finalize_turn()` 和 `repair_partial_commit()`
- 在提交成功后推进 `latest-turn.json`
- 只在 stable working set 存在时推进 `latest-working-set.json`

### 6.2 AgentRunner

`AgentRunner` 不直接写 `TurnState`。它只通过结构化回调把以下信息交给外层：

- 已声明 tool calls
- 已完成 tool result
- 已持久化 artifact

### 6.3 AutoCompact

`AutoCompact` 的职责是为 idle 会话生成候选 working set，而不是改写运行时主状态：

- 只读取最新 committed turn 及其 manifest
- 生成 `published_by=autocompact_candidate` 的 snapshot
- 写 `autocompact-candidate` 索引
- 不推进 `latest-working-set.json`

### 6.4 Dream

`Dream` 当前消费 committed turn 的结构化输入：

- `TurnCapsule`
- `WorkingSetSnapshot`
- artifact digest/render
- candidate signals

`Dream` 在自己的文件上下文里明确写有：

- `working/CURRENT.md` 是 mirror-only output
- 不把它当作 source of truth

## 7. CURRENT.md 与归档文件的现状

### 7.1 working/CURRENT.md

`working/CURRENT.md` 目前仍可能出现在 system prompt 中，但它的角色已经降级为：

- 人类可读的 mirror/handoff 视图
- Dream 可更新的镜像输出
- 与结构化状态并存的兼容文本层

它不再承担以下职责：

- 运行时短期状态唯一来源
- 恢复判定依据
- Dream 当前短期状态推断入口

### 7.2 archive/history.jsonl

`archive/history.jsonl` 继续保留，但当前职责是：

- 归档与检索
- Consolidator 的历史沉淀输出
- 长期记忆流程的辅助材料

它不再承担：

- prompt 主输入
- 当前 turn 状态判断
- Dream 对当前短期状态的主推断入口

### 7.3 session.messages 与 last_consolidated

原始会话日志和 `last_consolidated` 仍存在，当前职责是：

- 保存原始交互与兼容会话存储
- 为 `Recent Raw Turns` 提供最近合法尾部
- 为历史压缩和归档提供原始材料

它们不再承担短期运行时主工作内存职责。

## 8. 已删除或已降级的旧路径

下列路径已经退出短期上下文主链：

- 以 `session.messages` 尾部作为默认短期主输入
- 以 tool message 截断结果作为主要证据保留策略
- 以 `working/CURRENT.md` 作为运行时主状态
- 以 `CURRENT.md + history.jsonl` 反推当前短期状态
- 以 idle summary 注回作为短期主连续性机制

仍然保留但已降级的内容：

- `session.messages`：原始日志
- `last_consolidated`：归档边界与兼容字段
- `working/CURRENT.md`：镜像/交接视图
- `archive/history.jsonl`：归档/检索输入

## 9. 对外结论

如果要回答“nanobot 现在的短期上下文到底看什么”，答案是：

- 稳定主状态看 `WorkingSetSnapshot`
- 恢复和完成判定看 `TurnState + CommitManifest`
- 最近细节补充看 `Recent Raw Turns`
- 可复用证据和结论看 `ArtifactRecord + TurnCapsule`

不要再把 `CURRENT.md`、`history.jsonl` 或原始消息尾部当成当前主真相源。
